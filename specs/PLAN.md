## Source files (in build order)

### 1. `src/text_fx/exceptions.py`
- Purpose: All custom exception classes for the library
- Depends on: nothing
- Public: `TextFxError`, `UnknownEffectError` (with fuzzy-match suggestion field), `EffectMappingError`, `EngineError`, `CompositorError`, `TypographyError`, `ZeroDurationError`

### 2. `src/text_fx/easing/__init__.py`
- Purpose: Package init; re-exports easing callables
- Depends on: `easing/curves.py`
- Public: `get_easing`, `EASING_NAMES`

### 3. `src/text_fx/easing/curves.py`
- Purpose: All easing functions as pure float→float callables; used by every engine
- Depends on: `numpy`
- Public: `linear`, `ease_in`, `ease_out`, `ease_in_out`, `ease_out_cubic`, `ease_out_back`, `ease_out_elastic`, `spring`, `bounce`, `get_easing(name) -> Callable`

### 4. `src/text_fx/typography/__init__.py`
- Purpose: Package init; re-exports primary typography API
- Depends on: `typography/fonts.py`, `typography/renderer.py`, `typography/layout.py`
- Public: `render_text`, `load_font`, `compute_layout`

### 5. `src/text_fx/typography/fonts.py`
- Purpose: Font loading, system font discovery, path resolution; optional fonttools integration
- Depends on: `Pillow`, optionally `fonttools`; stdlib `pathlib`, `warnings`
- Public: `load_font(family, size, weight) -> ImageFont`, `find_system_font(family) -> Path | None`, `list_system_fonts() -> list[str]`

### 6. `src/text_fx/typography/layout.py`
- Purpose: Multi-line text layout; word and character splitting for per-unit animation
- Depends on: `Pillow`, `typography/fonts.py`
- Public: `compute_layout(text, font, max_width) -> LayoutResult`, `split_words(text) -> list[str]`, `split_chars(text) -> list[str]`; dataclass `LayoutResult` (lines, word_rects, char_rects, total_size)

### 7. `src/text_fx/typography/renderer.py`
- Purpose: PIL-based text rendering to RGBA Pillow Image at target font + style (the typography pass)
- Depends on: `Pillow`, `numpy`, `typography/fonts.py`, `typography/layout.py`
- Public: `render_text(text, config) -> Image.Image` (RGBA), `render_text_layers(text, config) -> list[Image.Image]` (one per character/word for per-unit engines)

### 8. `src/text_fx/engines/base.py`
- Purpose: Abstract base class `Engine`; defines the render_effect() contract; frame-sequence protocol
- Depends on: `abc`, `numpy`, `Pillow`, `exceptions.py`
- Public: `Engine` (ABC) with `render_effect(text_image, params, config, output_path) -> Path`, `render_frames(text_image, params, config) -> Iterator[np.ndarray]`; dataclass `EngineParams`; `FrameSequence` type alias

### 9. `src/text_fx/engines/alpha/__init__.py`
- Purpose: Alpha bucket package init; registers and re-exports all three alpha engines
- Depends on: `alpha/fade.py`, `alpha/typewriter.py`, `alpha/cascade.py`
- Public: `FadeEngine`, `TypewriterEngine`, `CascadeEngine`

### 10. `src/text_fx/engines/alpha/fade.py`
- Purpose: Base engine for all alpha-fade variants: fade_in, fade_out, cross_dissolve_text, hard_cut_title, dip_to_black_title, dip_to_white_title, blur_fade_in, blur_fade_out
- Depends on: `engines/base.py`, `easing/curves.py`, `numpy`, `Pillow`
- Public: `FadeEngine(Engine)` — params: `direction` (in|out|dissolve|hard|dip_black|dip_white), `blur_start`, `blur_end`, `easing`

### 11. `src/text_fx/engines/alpha/typewriter.py`
- Purpose: Base engine for character-sequential reveal: typewriter, cursor_typewriter, backspace_delete, word_reveal, line_reveal
- Depends on: `engines/base.py`, `easing/curves.py`, `typography/layout.py`, `numpy`, `Pillow`
- Public: `TypewriterEngine(Engine)` — params: `unit` (char|word|line), `direction` (forward|backward), `cursor` (bool), `cursor_blink_hz`

### 12. `src/text_fx/engines/alpha/cascade.py`
- Purpose: Base engine for staggered-unit fade reveals: character_fade_cascade, word_fade_cascade
- Depends on: `engines/base.py`, `easing/curves.py`, `typography/layout.py`, `numpy`, `Pillow`
- Public: `CascadeEngine(Engine)` — params: `unit` (char|word), `stagger_s`, `easing`

### 13. `src/text_fx/engines/kinetic/__init__.py`
- Purpose: Kinetic bucket package init; registers and re-exports all five kinetic engines
- Depends on: `kinetic/slide.py`, `kinetic/spring.py`, `kinetic/bounce.py`, `kinetic/spin.py`, `kinetic/oscillate.py`
- Public: `SlideEngine`, `SpringEngine`, `BounceEngine`, `SpinEngine`, `OscillateEngine`

### 14. `src/text_fx/engines/kinetic/slide.py`
- Purpose: Base engine for translational motion: slide_left_text, slide_right_text, slide_up_text, slide_down_text, drop_in, rise_up, impact_slam (large fast slide + jitter)
- Depends on: `engines/base.py`, `easing/curves.py`, `numpy`, `Pillow`, `cv2`
- Public: `SlideEngine(Engine)` — params: `direction` (left|right|up|down), `overshoot` (bool), `impact_frames`, `easing`

### 15. `src/text_fx/engines/kinetic/spring.py`

### 16. `src/text_fx/engines/kinetic/bounce.py`

### 17. `src/text_fx/engines/kinetic/spin.py`

- Purpose: Code-gen script (NOT shipped in the wheel; run at development time) that classifies all 120 catalog slugs into engine mappings and writes `src/text_fx/data/effect_mapping.json`
- Depends on: `stdlib json`, `pathlib`
- Logic:
  - `CATEGORY_TO_ENGINE`: maps category string to default base engine name
  - `NAME_TO_BASE`: per-slug override table for slugs whose best engine differs from category default

- Output schema per entry:

  ```json
    "slug": "neon_glow_text",
    "base_engine": "glow",
    "engine_bucket": "neon",
    "params": {"color": "#FF00FF", "blur_radius": 24, "intensity": 1.5, "mode": "glow"},
    "prefer_engine": "auto"
  }
  ```

## Build order rationale

The dependency graph flows strictly bottom-up:

1. **`exceptions.py`** — no imports from this package; everything else can import it
2. **`easing/curves.py`** — pure math, only numpy; all engines depend on it
3. **`typography/fonts.py`** — no internal deps; engines need fonts before they can render text
4. **`typography/layout.py`** — needs fonts; typewriter/cascade/oscillate engines need layout
5. **`typography/renderer.py`** — needs fonts + layout; produces the RGBA image that every engine consumes
6. **`engines/base.py`** — depends on easing + exceptions only; defines the ABC all engines implement
7. **All 25 engine files** (steps 7–32) — each depends only on base.py, easing, typography, numpy, Pillow, cv2; no engine depends on another engine
8. **All 6 engine bucket `__init__.py`** — aggregate imports after all files in the bucket exist
9. **`engines/__init__.py`** — aggregates all bucket inits; builds `ENGINE_REGISTRY`
10. **`catalog/loader.py`** — needs only stdlib + importlib.resources
11. **`catalog/mapping.py`** — needs only stdlib + importlib.resources
12. **`catalog/__init__.py`** — aggregates loader + mapping
13. **`config.py`** — needs pydantic + exceptions
14. **`resolver.py`** — needs catalog, engines, config, exceptions
15. **`compositor.py`** — needs ffmpeg-python, config, exceptions
16. **`api.py`** — needs everything above
17. **`schema/generator.py`** — needs catalog + config; no circular deps
18. **`schema/parser.py`** — needs config + catalog + resolver
19. **`schema/__init__.py`** — aggregates generator + parser
20. **`cli.py`** — depends on api + schema + catalog (top of the tree; nothing depends on it)
21. **`__init__.py`** — depends on api + config + catalog + schema; written last so version string is the only missing piece

`scripts/generate_mapping.py` is a dev-time tool; it reads `effects_catalog.json` and writes `effect_mapping.json`. It must be run **before** `uv build` so the data file is present for the wheel. It has no runtime dependencies on the package itself.

## Open questions / spec ambiguities

- The spec's §3.1 table lists 3 base engines for `basic_editing` (alpha, typewriter, cascade) but the spec structure shows only `fade.py` and `typewriter.py` in the layout — `cascade.py` is present in the task description. This plan treats `cascade.py` as the third alpha engine. No blocking issue.

- The `3d_rotate_title` slug contains a leading digit which is invalid as a Python identifier. The engine dispatch routes it to `SpinEngine` with `perspective_approx=True`. The slug string itself is never used as a Python identifier, only as a dict key. No blocking issue.

- The spec mentions `holographic_text` as a web_overlay candidate in §3.4, but the catalog uses `hologram_text` as the slug. This plan uses `hologram_text` (matching the catalog) throughout. The `PREFER_ENGINE_OVERRIDES` table in `generate_mapping.py` should key on `hologram_text`.

- The spec lists `apply_text_sequence` in §2.3 and `render_overlay` in §2.4 but does not list them in the `__all__` example. This plan includes both in `__all__` per the rule that every public function in §2 must be exported.

- CLI dependencies (`typer`, `rich`) are not listed in §4. They should be added to `[project.dependencies]` in `pyproject.toml`. Suggest adding `typer >= 0.9` and `rich >= 13` as required deps (or as a `cli` extra if a lightweight install is desired). No blocking issue for PLAN.md, but the implementer must resolve this before writing `pyproject.toml`.
